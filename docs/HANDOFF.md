# HANDOFF — pointer to latest session

The current session handoff lives in `docs/HANDOFFS/`.

**Latest:** `docs/HANDOFFS/2026-06-26-session-01-complete.md`

## How to use this in a new chat

1. Open the latest file from `docs/HANDOFFS/` (filenames are date + session number, sort chronologically).
2. Paste the full contents at the start of the new chat.
3. The new chat will ask you to paste the verification snippet (git status, geant4-config, etc.) before any code is written.
4. At the end of the session, the chat generates the next handoff file and updates the `Latest:` pointer above.

## Why one file per session

A thesis project benefits from a chronological archive of "what was decided when." Single-file rolling handoffs lose history. The per-session archive is also a learning journal — useful for thesis Chapter 5 ("Construction of the Simulation Framework"), and for supervisors/examiners who want to see the build-up.

The pointer file (this one) gives any new chat exactly one canonical path to grab, no ambiguity about which handoff is current.
